angular的安全性

了解angular的security(安全)政策可以帮助我们在开发的过程中避免引入一些不安全的因素为project留下安全的隐患，
那么不安全的因素有哪些以及angular如何协助我们去避免这些问题呢？接下来给大家讲解几点：

  • XSS(cross-site scripting)页面攻击
  • http层面的攻击

阻止XSS(cross-site scripting)攻击

攻击者注入脚本到web页面来窃取用户的数据，比如登录的用户名密码等等，对于这种攻击
我们的主要做法是阻止攻击者的数据进入到我们的DOM。
angular具备这样的安全模块
angular默认是不信任任何的值或者数据的，当一个来源于模板，属性，样式等的值插入到DOM，angular会做处理以及清除掉这些数据
angular的模板本身是完全受到信任的，所以要避免攻击者直接修改到模板的代码，主要手段就是使用
离线模板编译(https://angular.io/guide/security#offline-template-compiler)

直接使用DOM的api

不要使用第三方的api，没办法去保证里面的安全性

CSP(Content Security Policy)

非常有效的技术去阻止XSS攻击(http://www.html5rocks.com/en/tutorials/security/content-security-policy/)

使用离线的模板编译

服务器端的XSS保护

信任安全的数据

如果开发的过程中不得不加入某一个值，为了避免被angular自动给清理掉，可以自己先做好声明

  • bypassSecurityTrustHtml
  • bypassSecurityTrustScript
  • bypassSecurityTrustStyle
  • bypassSecurityTrustUrl
  • bypassSecurityTrustResourceUrl

http层面的攻击

伪造跨站的请求CSRF或者XSRF

需要客户端和服务器端合作来阻止，好的方式是服务器随机产生token给客户端的cookies，每次客户端的请求都要携带token到后台去做验证，
如果token不存在或者不符合，就直接拒绝。
