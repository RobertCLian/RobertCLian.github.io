我的开发环境 | GIT

Posted on 2019-05-23 | In web前端

团队开发在代码的管理上面，不同公司可能有不同的一些方案，我们公司现在使用的比较流行的GIT来管理。

使用GIT的童鞋或多或少遇到了一些苦恼的问题，有时也浪费时间，我想分享一下我的git的一些使用心得。

  1. 考虑到代码分析与合并可能带来的问题，我们并不是直接在local repository直接修改代码，而是另外拷贝一份import到eclipse， 如下图

可以看到我们多了一个代码compare的动作，这样做有几个好处：

  1. 使用WinMerge软件可以很直观地看到修改的代码的部分，提交之前可以自己做一次code review。
  2. 可以极大的避免提交不必的代码到Remote从而引起编译错误或其它问题。
  3. 提交代码冲突local被覆盖的时候，新的修改在eclipse的folder还在。

GIT常用到的几个命令

1. 拿远程某个分支的更新
git pull

1. 拿远程某个分支的更新并保留本地的修改
git pull --rebase

3. 查看GIT的日志
git log

4. 查看GIT的修改状态
git status

5. 添加所有修改的文件
git add *

6. 添加要修改的文件
git add *

7. 提交的comment
git commit -m "your comment"

8. 修改commit但是还没有push的comment
git commit --amend

9. 把本地的修改提交到remote branch
git push

10. 比较文件的修改部分
git diff <file>

11. 恢复GIT到之前的某个版本
git reset --hard <commit>
